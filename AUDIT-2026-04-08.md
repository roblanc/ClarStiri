# ClarStiri — Full Project Audit

**Date:** 2026-04-08 | **Stack:** React 18.3, TypeScript 5.8, Vite 5.4, Supabase/Vercel

---

## Architecture

Well-structured. Clean separation: 11 pages, 15 domain components + 62 shadcn/ui primitives, 4 hooks, 2 service layers. Data flows sensibly: RSS sources (55) → parallel fetch → Jaccard/entity clustering → Redis cache → TanStack Query → localStorage fallback.

Cron job runs hourly from Vercel (Frankfurt), refreshes Redis. Client has a 3-tier fallback chain: API → localStorage → client-side aggregation. This is robust.

The aggregation algorithm (hybrid Jaccard + entity + bigrams, threshold 0.32, 7-day window) is the intellectual core. It works, but the O(n²) comparison in `findSimilarStories()` will scale poorly past ~200 articles.

---

## Code Quality

### Critical

**TypeScript is effectively disabled.** `tsconfig.json` has `noImplicitAny: false`, `strictNullChecks: false`, `noUnusedLocals: false`. You're writing JavaScript with extra steps. This is the single highest-impact thing to fix — it would surface real bugs immediately.

### Medium

**Console.logs in production.** `newsService.ts` lines 58-66, `useNews.ts` line 59, `shareButton.tsx`. Replace with Sentry or at minimum strip in build.

**Error masking.** `useAggregatedNews()` catches timeouts and falls to cache silently. If both API and cache fail, user sees infinite spinner — no error state rendered.

**`fetchOgImage()`** in `aggregation.ts` (lines 19-21) swallows all errors in a bare catch block.

**Relative imports.** `shareButton.tsx` imports from `'../../shared/htmlEntities'` — fragile; use the `@/` alias.

### Low

Unused `rss-parser` in `package.json` (you parse RSS manually via DOMParser). Font duplication between `@fontsource` packages and custom `/fonts/` directory. `htmlEntities.ts` double-pass decoding is overkill for most RSS feeds.

## Performance

### Good

Vite chunk splitting is well-configured (6 vendor bundles). ES2020 target. Image optimization via wsrv.nl with WebP + responsive srcset. Lazy routes via `React.lazy()`. Font preloading.

### Concerns

**Radix UI: 99 KB vendor chunk.** 17 primitives imported — audit which are actually rendered. `navigationMenu`, `commandPalette`, `resizable` look suspect.

**`useTextFit()` triggers layout measurement on every render.** The font config object is created inline in `NewsCard.tsx` (not memoized), forcing recalculation.

**`useMemo` in `useAggregatedNews()`** depends on `query.data` which changes on every TanStack Query state transition (stale/loading/error), not just on actual data change.

**No service worker.** Offline = broken, despite having localStorage cache that could serve.

## SEO

### Implemented

Meta tags (OG, Twitter Card), `react-helmet-async` per page, `robots.txt`, build-time `sitemap.xml`, security headers in `vercel.json` (CSP, HSTS, X-Frame-Options). Solid baseline.

### Missing

No per-story OG images (static fallback only). No canonical tags. No JSON-LD `NewsArticle` structured data (the `NewsSchema.tsx` component exists but unclear if wired up). Sitemap doesn't include dynamic story URLs. ClarStiri consumes 55 RSS feeds but doesn't produce one of its own — missed discoverability.

---

## UX / Design / Accessibility

### Good

Mobile-first Tailwind responsive design with separate mobile/desktop card layouts. Dark mode with system preference detection. Skeleton loading states. Semantic HTML (`<article>`, `<nav>`, `<header>`).

### Accessibility Gaps (WCAG 2.1)

No `aria-label` on icon-only buttons (theme toggle, share, search). No skip-to-main-content link. Bias visualization bars lack `role="img"` + alt text. Search input has no associated `<label>`. No visible focus indicators confirmed. Color contrast on `text-muted-foreground` and bias colors (blue/purple/red) may fail WCAG AA 4.5:1 ratio — needs testing.

---

## Production Readiness

### Secure

API keys not hardcoded. Security headers configured. Image proxy has SSRF protection. Redis credentials in env vars.

### Risks

**CRON_SECRET:** If missing in production, `/api/cron/refresh-news` is publicly accessible without auth. Should fail-closed. Currently permissive.

**No `.env.example`:** New contributors have no idea what env vars are needed.

**No error tracking.** Console-only logging. No Sentry, no LogRocket, no structured logging.

**Redis 14-day inactivity wipe** (Upstash free tier). No monitoring or alerting if this happens. Silent data loss.

**No error boundary per route** — only a global `AppErrorBoundary`. One bad story page takes down the whole app.

---

## Dependencies

393 packages total. `recharts` (heavy, only used on Barometer page) and `framer-motion` (large, minimal usage) are the main bloat candidates. `embla-carousel-react` — verify it's used. `rss-parser` — unused, remove.

No known vulnerabilities as of audit date.
